# ESG Sustainability Analytics — S&P 500 Companies

End-to-end ESG (Environmental, Social, Governance) analytics on a Kaggle dataset of **866 sustainability reports from 263 S&P 500 companies (2014–2023)**. The project takes a pre-scored ESG text dataset and builds a full analytics + machine-learning layer on top of it — cleaning, validation, feature engineering, EDA, a company risk-index, and a leakage-safe next-year score forecaster.

> **Status:** Steps 1–6 complete (this repo). Steps 7 (GenAI RAG copilot) and 8 (Streamlit dashboard) are planned.

## ▶️ How to run
1. Download `preprocessed_content.csv` from Kaggle: *"ESG Sustainability Reports of S&P 500 Companies"* and place it next to the notebook.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Open and run all cells:
   ```bash
   jupyter notebook ESG_Methodology_Steps_1-6.ipynb
   ```
   This regenerates `esg_features.csv`, `esg_risk.csv`, the `figures/`, and the trained model.

## 🔑 Methodology at a glance
| Step | What | Result |
|---|---|---|
| 1 · Clean | drop index col, safe-parse entities, fix types, dedup ticker+year | 864 × 9 |
| 2 · Validate | verify `total ≈ e+s+g`, ranges, missing | *approximately* additive (coeffs 1,1,1, R²=0.999), tolerance-checked |
| 3 · Features | text / composition / leakage-safe lag features | 27 columns (18 engineered) |
| 4 · EDA | 4 targeted charts + insights | Social highest, Environmental lowest; disclosure rising |
| 5 · Risk Index | 0.40·E + 0.30·S + 0.30·G, ranked | 263 companies ranked |
| 6 · Forecast | leakage-safe regression | Linear Regression, **R² 0.913**, beats naive baseline |

## ✅ Key engineering decisions
- **No data leakage:** the forecaster uses only past/structural features (`shift(1)` lag features). Current-year `e/s/g/total` and `yoy_change` are deliberately excluded, giving an honest **R² ≈ 0.91** instead of a leakage-inflated 1.0.
- **Assumptions verified, not trusted:** the `total = e+s+g` identity is only *approximately* true (~22% of rows deviate beyond rounding), so it is handled with tolerance-based comparison and outlier flagging.
