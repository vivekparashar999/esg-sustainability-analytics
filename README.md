# ESG Sustainability Analytics — S&P 500 Companies

End-to-end ESG (Environmental, Social, Governance) analytics on a Kaggle dataset of 866 sustainability reports from 263 S&P 500 companies (2014-2023). The dataset comes pre-scored (text, entities and E/S/G scores are given); this project builds the analytics and machine-learning layer on top of it — cleaning, validation, feature engineering, EDA, a company risk-index, and a leakage-safe next-year score forecaster.

Status: Steps 1-6 complete.

A full methodology write-up is in [REPORT.md](REPORT.md).

## Repository structure

```
ESG_Methodology_Steps_1-6.ipynb   main notebook, Steps 1-6 (runs top-to-bottom)
REPORT.md                         methodology report (Steps 1-6)
figures/                          EDA charts
outputs/models/esg_model.pkl      trained forecasting model
requirements.txt                  Python dependencies
```

The large CSVs (`preprocessed_content.csv`, `esg_features.csv`, `esg_risk.csv`, ~88 MB each) are not committed. The input CSV is downloaded from Kaggle; the feature and risk CSVs are regenerated by the notebook.

## How to run

1. Download `preprocessed_content.csv` from Kaggle ("ESG Sustainability Reports of S&P 500 Companies") and place it next to the notebook.
2. Install dependencies: `pip install -r requirements.txt`
3. Run all cells in `ESG_Methodology_Steps_1-6.ipynb`. This regenerates the feature/risk CSVs, the figures, and the trained model.

## Methodology summary

| Step | What | Result |
|------|------|--------|
| 1. Clean | drop index column, safe-parse entities, fix types, remove duplicate ticker+year | 864 x 9 |
| 2. Validate | verify total ≈ e+s+g, ranges, missing values | approximately additive (coeffs 1,1,1, R²=0.999), tolerance-checked |
| 3. Features | text, composition and leakage-safe lag features | 27 columns (18 engineered) |
| 4. EDA | 4 charts with insights | Social highest, Environmental lowest; disclosure rising over time |
| 5. Risk Index | 0.40 E + 0.30 S + 0.30 G, normalized and ranked | 263 companies ranked |
| 6. Forecast | leakage-safe regression | Linear Regression, R² 0.913, beats naive baseline |

## Key decisions

- No data leakage: the forecaster uses only past and structural features (shift(1) lag features). Current-year e/s/g/total and yoy_change are deliberately excluded, giving an honest R² of about 0.91 instead of a leakage-inflated 1.0.
- Assumptions are verified, not assumed: the `total = e+s+g` relationship is only approximately true (about 22% of rows differ by more than rounding), so it is handled with a tolerance check and outlier flagging.
